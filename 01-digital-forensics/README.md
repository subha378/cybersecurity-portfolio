# Digital Forensics Investigation — Seized Laptop Case Study

## Overview

This project involved conducting a digital forensic investigation of a seized Windows laptop image using Autopsy. The goal was to identify relevant evidence, document findings clearly, and apply ethical and legal considerations when handling digital evidence.

## Objectives

* Understand chain of custody in a digital forensics scenario
* Examine evidence integrity and possible tampering concerns
* Identify active user accounts and login activity
* Analyse connected USB and physical devices
* Review web searches, installed software and user activity
* Identify unique video files
* Consider ethical handling of Personally Identifiable Information (PII)

## Tools Used

* Autopsy

## Skills Demonstrated

* Digital forensic investigation
* Windows artifact analysis
* Chain of custody awareness
* Evidence recovery
* USB device artifact analysis
* Web history analysis
* Installed program analysis
* PII handling and ethics
* Investigation reporting
* Technical documentation

## Key Findings Summary

The investigation involved analysing operating system accounts, connected USB devices, browser activity, installed applications, unique media files and personal information artifacts. The findings were documented in a structured forensic report with evidence locations, acquisition steps and explanations of how the results were obtained.

## What I Learned

This project helped me understand how forensic investigators collect, analyse and report digital evidence in a careful and ethical way. It also strengthened my ability to use forensic tools, interpret system artifacts, document technical findings and consider privacy obligations during investigations.
